# DRIVE Stage1 训练问题排查与解决报告

## 项目概述

- **目标**：使用 OpenRLHF (v0.10.1) 的 GRPO 算法对 Qwen2.5-Coder-3B-Instruct 进行代码生成强化学习训练
- **硬件**：4× NVIDIA RTX A6000 (48GB each)，单节点
- **数据**：1000 条编程题（`stage1_base_str.jsonl`），含 prompt + test_cases
- **奖励**：自定义 Flask 服务（`reward_service.py`），通过 `--remote_rm_url` 接入，基于代码执行正确性 + 格式分

---

## 一、脚本参数问题（共 5 个）

### 1.1 参数名错误
- `--dataset` → `--prompt_data`
- `--bf16` → `--param_dtype bf16`
- `--kl_coef` → `--init_kl_coef`
- `--enable_tensorboard` + `--tensorboard_log_dir` → `--use_tensorboard <path>`

### 1.2 数据集字段名不匹配
- `--label_key label` → `--label_key test_cases`（匹配数据集实际字段）

### 1.3 GRPO 要求 n_samples_per_prompt > 1
- 添加 `--n_samples_per_prompt 8`

### 1.4 数据集 schema 不一致导致 pyarrow 解析失败
- **错误**：`ArrowInvalid: Column(/test_cases/[]/input/[]) changed from string to array`
- **原因**：`test_cases` 字段中 `input` 类型不一致（有的是 string list，有的是 dict list）
- **解决**：将 `test_cases` 序列化为 JSON 字符串存入新数据集 `stage1_base_str.jsonl`

### 1.5 DeepSpeed batch size 约束
- **错误**：`train_batch_size != micro_batch * grad_acc * world_size`
- **解决**：确保 `train_batch_size` 能被 `micro_train_batch_size × world_size` 整除

---

## 二、GPU 资源调度问题（共 3 个）

### 2.1 Ray 调度卡死
- **配置**：actor=3 GPU + vllm=1 GPU = 4 GPU 全占
- **问题**：PPOTrainer 主进程无法获得 GPU 资源，无限等待
- **解决**：actor 降为 2 GPU，留 1 GPU 给 PPOTrainer

### 2.2 ReferenceModelActor world_size=0
- **配置**：`ref_num_nodes=0, ref_num_gpus_per_node=0` 但 `init_kl_coef > 0`
- **问题**：KL loss 需要 ref model，但未分配 GPU
- **解决**：`--ref_num_nodes 1 --ref_num_gpus_per_node 2 --colocate_actor_ref`，让 ref model 与 actor 共享 GPU

### 2.3 最终 GPU 分配方案
| 角色 | GPU 数量 | 说明 |
|---|---|---|
| Actor + Ref (colocate) | 2 | ZeRO-3 分片，共享 placement group |
| vLLM Rollout | 1 | 30% 显存利用率 |
| PPOTrainer | 1 | 主进程 |

## 三、OOM 问题（共 3 次）

### 3.1 第一次 OOM — backward 阶段
- **错误**：`Tried to allocate 1.16 GiB, only 339 MiB free`
- **原因**：actor + ref 共享 GPU，backward 时激活值占满显存
- **措施**：
  - `micro_train_batch_size 2 → 1`
  - 添加 `--gradient_checkpointing`

### 3.2 第二次 OOM — optimizer step 阶段
- **错误**：`Tried to allocate 5.75 GiB, only 5.50 GiB free`
- **原因**：ZeRO-2 梯度聚合需要完整梯度分区（~6GB）
- **措施**：`--zero_stage 2 → 3`（参数/梯度/优化器全分片）

### 3.3 第三次 OOM — 训练启动阶段（今天最后一次）
- **错误**：`Tried to allocate 22 MiB, only 35 MiB free on GPU 0`
- **原因**：上次训练的残留进程（PID 1358539）占用 GPU 0 的 38GB 显存
- **措施**：需要手动 `kill -9` 残留进程后重新启动
- **状态**：尚未执行，明天需要先清理 GPU 再启动训练

## 四、remote_rm_url 协议适配问题

### 4.1 OpenRLHF 的 API 协议
- **请求**：`POST {"query": [str], "prompts": [str], "labels": [str]}`（每次单条）
- **响应解析**（`agent.py:292-299`）：
  - 优先取 `scores`，fallback 到 `rewards`
  - 期望标量值，不是列表

### 4.2 reward_service.py 的响应格式修复
- **问题**：最初返回 `{"scores": [0.5]}`（列表），导致 `reward_val=None`，触发 `reward_model_group is required`
- **解决**：改为返回 `{"rewards": score, "scores": score}`（标量）

### 4.3 函数调用型测试用例支持
- **问题**：77% 的数据是函数调用型（`input=[*args], output=any`），原 `execute_code` 只支持 stdin/stdout
- **解决**：在 `reward_service.py` 中新增 `_execute_func` 函数，提取代码中的函数名，通过 JSON stdin 传参调用

## 六、当前 train_stage1.sh 最终配置

```bash
python -m openrlhf.cli.train_ppo_ray \
    --ref_num_nodes 1 \
    --ref_num_gpus_per_node 2 \
    --colocate_actor_ref \
    --reward_num_nodes 0 \
    --reward_num_gpus_per_node 0 \
    --critic_num_nodes 0 \
    --critic_num_gpus_per_node 0 \
    --actor_num_nodes 1 \
    --actor_num_gpus_per_node 2 \
    --vllm_num_engines 1 \
    --vllm_tensor_parallel_size 1 \
    --vllm_gpu_memory_utilization 0.3 \
    --pretrain /mnt/sdc/ubuntu/cjz_projects/models/Qwen2.5-Coder-3B-Instruct \
    --save_path ./checkpoints/stage1_v2 \
    --input_key prompt \
    --label_key test_cases \
    --prompt_data /mnt/sdc/ubuntu/cjz_projects/DRIVE/data/stage1_base_str.jsonl \
    --input_template "{}" \
    --max_samples 1000 \
    --micro_train_batch_size 1 \
    --train_batch_size 128 \
    --rollout_batch_size 16 \
    --n_samples_per_prompt 8 \
    --max_new_tokens 768 \
    --max_epochs 1 \
    --zero_stage 3 \
    --gradient_checkpointing \
    --param_dtype bf16 \
    --advantage_estimator group_norm \
    --use_kl_loss \
    --init_kl_coef 0.01 \
    --remote_rm_url http://localhost:5000/scores \
    --stop_properly_penalty_coef 0.0 \
    --overlong_buffer_len 256 \
    --overlong_penalty_factor 1.0 \
    --use_tensorboard ./checkpoints/stage1_v2
```
