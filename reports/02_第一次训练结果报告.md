## 训练阶段分析

### 阶段一：学习期（Step 1-16）
- 奖励从 0.051 稳步上升至 0.091（+78%）
- 截断率稳定在 42-47%
- `group_reward_std` 从 0.044 上升至 0.065，GRPO 信号增强
- 模型在学习生成有效代码

### 阶段二：长度 hacking 期（Step 17-39）
- 截断率从 43% 飙升至 88%
- `response_length` 从 541 暴涨至 1565
- `group_reward_std` 从 0.065 暴跌至 0.019，GRPO 信号消失
- 模型学会了"生成尽可能长的文本"，所有样本都撞到 max_len
- 奖励开始下降（0.098 → 0.092）

### 阶段三：崩溃期（Step 40-62）
- 截断率维持在 90%
- `policy_loss` 居高不下（1.8-3.8）
- `return` 持续为负（KL 惩罚主导）
- 模型陷入均匀坏策略，无法恢复

## 核心问题：长度 hacking

模型在训练中期发现"生成更长的回复"可以偶尔获得更高的奖励（更多代码 = 更多机会通过测试），但超过 max_len 后被截断，代码不完整反而得 0 分。这形成恶性循环：
1. 模型生成更长 → 偶尔得分更高
2. 但大部分被截断 → 得 0 分
3. 组内所有样本都被截断 → group_reward_std → 0 → GRPO 无信号
4. 策略无法更新 → 卡在坏策略
