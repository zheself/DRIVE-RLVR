## 三、v3 训练分析

### 3.1 ❌ 问题：v3 效果不如 v2

**核心发现**：v3 从 v2 checkpoint 继续训练后，性能反而下降了。

**具体表现**：
1. **score 下降**：从 v2 的 0.094 降到 0.087（-7.4%）
2. **截断率上升**：从 11.7% 升到 14.1%
3. **代码变长**：从 364 token 增加到 426 token
4. **reward 变负**：从 0.065 降到 -0.017

### 3.2 ✅ 正面现象：GRPO 信号增强

**唯一的改善**：
- **group_reward_std 大幅提升**：从 0.060 增加到 0.175（+192%）
- 说明 temperature=1.2 成功增加了采样多样性
- GRPO 的组内对比信号更强了

### 3.3 🤔 可能的原因分析

#### 原因 1：学习率过小导致训练不充分
- actor_learning_rate = 5e-7 可能太保守
- 模型参数更新幅度太小，无法有效学习
- 虽然 policy_loss 在下降（0.669），但 score 没有提升

#### 原因 2：temperature=1.2 引入了过多噪声
- 虽然增加了多样性（group_reward_std 提升），但也可能引入了低质量样本
- 模型在学习这些噪声样本时，反而破坏了 v2 已经学到的能力

#### 原因 3：只跑了 1 个 epoch
- 训练日志显示只完成了 62 步（1 个 epoch）
- 原计划是 3 个 epoch（189 步），但训练提前结束了
- 可能是因为数据集遍历完就停止了（`Prompt dataloader is exhausted`）

#### 原因 4：从 checkpoint 继续训练的策略问题
- v2 已经在 1 个 epoch 内收敛到较好状态
- 继续训练可能导致过拟合或遗忘（catastrophic forgetting）
- 特别是在小数据集（1000 条）上多轮训练

## 四、训练过程观察

### 4.1 训练曲线特征

1. **score 先升后降**：
   - Step 1-51：从 0.071 稳步上升到 0.091（峰值）
   - Step 51-62：从 0.091 下降到 0.087
   - 出现了轻微的过拟合迹象

2. **截断率波动**：
   - 在 11.7%-28.9% 之间波动
   - 没有像 v2 那样稳定在低位

3. **response_length 不稳定**：
   - 从 485 降到 367，再升到 426
   - 说明模型对输出长度的控制不够稳定

4. **group_reward_std 持续较高**：
   - 始终保持在 0.12-0.18 之间
   - temperature=1.2 确实有效增加了多样性

### 4.2 训练终止原因

从日志可以看到：

- 数据集只遍历了 1 遍就结束了
- 虽然设置了 `max_epochs=3`，但实际只跑了 1 个 epoch

## 五、结论与建议

### 5.1 v3 训练结论

❌ **v3 训练失败**：从 v2 checkpoint 继续训练并没有带来性能提升，反而导致了退化。

**关键教训**：
1. 在小数据集（1000 条）上，1 个 epoch 可能已经足够
2. 从 checkpoint 继续训练需要更谨慎的超参数调整
3. temperature=1.2 虽然增加了多样性，但也可能引入了噪声

### 5.2 下一步建议

#### 方案 A：回退到 v2（推荐）
- **v2 是目前最好的模型**（score=0.094, truncated=11.7%）
- 使用 v2 checkpoint 进行后续工作

#### 方案 B：增加数据量，从头训练 v4
- 当前只有 1000 条数据，数据量太少
- 准备更多高质量编程题（目标 5000-10000 条）
- 从 Qwen2.5-Coder-3B-Instruct 原始模型开始训练
- 保持 v2 的成功配置（temperature=1.0, lr=1e-6, 1 epoch）

#### 方案 C：调整超参数重新训练 v4
如果坚持从 v2 继续训练，建议：
- 提高学习率：5e-7 → 1e-6 或 2e-6
- 降低 temperature：1.2 → 1.0 或 1.1
- 减少 epoch：3 → 1 或 2
- 增加 save_steps：20 → 10，更密集地保存 checkpoint

#### 方案 D：进入 Stage 2
- 不再优化 Stage 1 的基础训练
- 使用 v2 作为 Stage 1 的最终模型
- 准备更难的编程题数据集，进行 Stage 2 进阶训练
