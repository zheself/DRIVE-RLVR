## 四、Reward 评分逻辑修复

### 4.1 定位的三个 Bug

**Bug 1：代码提取正则匹配到 think 内容**

模型输出经常是 ` ```python\n<think>...\n```python\ndef code()...``` ``` `，原有的 lazy 正则提取到了 think 内容而非代码。v2 有 65% 的样本存在此问题。

**Bug 2：`_execute_func` 无法识别 class 方法**

`^def` 正则只匹配行首的 `def`，无法识别 `class Solution:` 内的缩进方法。数据集中 770/1000 题是函数调用型。

**Bug 3：stdin 型 output 格式错误 + 缺少 I/O 包装**

- `str(['TTTT'])` → `"['TTTT']"` 永远不匹配实际输出 `"TTTT"`
- 模型只生成函数定义，没有 `input()`/`print()`，stdin 型无法执行

### 4.2 修复内容

在 `reward_service.py` 中：

1. **新增 `_extract_code()` 函数**：智能配对 ` ```python ` 和 ` ``` `，优先选有 def/class 且无 think 的块
2. **修改 `_execute_func()`**：支持 class 方法识别（`ClassName().method(*_args)`），添加常用库自动 import
3. **修改 `_compute()`**：修正 stdin 型 expected_output 格式，自动包装 `print(func(input()))`

### 4.3 修复验证

单元测试（`tests/test_reward_fixes.py`）：所有 case 通过 ✓

v2 模型重新评测（20 样本）：

| 指标 | 修复前 | 修复后 |
|---|---|---|
| 平均总分 | 0.092 | **0.340** (+270%) |
| 平均执行分 | 0.000 | **0.250** |
| 满分样本 | 0/20 | **5/20** (25%) |
