## 五、结果分析

### 5.1 通用能力无退化（所有模型）

所有 5 个模型在通用 benchmark 上的表现高度一致，GRPO 训练未引入灾难性遗忘：

- MMLU：59.71 → 59.69 ~ 59.74（最大变化 +0.03，统计误差范围内）
- HellaSwag：67.86 → 67.86 ~ 67.91（几乎不变）
- TruthfulQA：46.44 → 46.33 ~ 46.36（最大变化 -0.11）
- ARC-Challenge：48.89 → 48.81 ~ 49.32（波动范围 ±0.5%）
- WinoGrande：65.43 → 64.48 ~ 65.43（Stage2 r64 略低，其余持平）

无论是 Stage1 全参数训练、还是 Stage2 的 LoRA 微调（包括 v2 清洗后版本），都没有破坏模型的知识储备和语言理解能力。

### 5.2 数学推理一致提升

GSM8K 在所有训练后模型上都有提升：
- Stage1 v5：67.32（-0.23，略低于 baseline，但在误差范围内）
- Stage2 r64 / v2 r16 / v2 r64：均为 67.93（+0.38）

这与 GRPO 训练强化逻辑推理能力的目标一致。Stage2 的三个变体在 GSM8K 上表现完全相同，说明数学推理能力的提升主要来自 Stage1 训练，Stage2 的不同配置对此没有额外影响。

### 5.3 HumanEval 下降的原因：评测协议不匹配

lm_eval 的 HumanEval 结果（baseline 70.73 → 训练后 65.85~67.68）与此前自定义评测的结果（baseline 79.27% → v5 79.27%）存在明显差异。这不是模型能力退化，而是评测协议差异导致的：

| | 自定义评测（报告18） | lm_eval 本次评测 |
|---|---|---|
| Prompt 格式 | 训练格式（中文指令 + `<think>` 格式要求） | 原始英文 docstring 直接补全 |
| Chat Template | 使用 | 未使用 |
| 代码提取 | 从 `<think>` + ` ```python ` 结构化输出中解析 | stop sequence 截断（`\ndef`、`\nclass` 等） |
| Baseline pass@1 | 79.27% | 70.73% |
| v5 pass@1 | 79.27% | 66.46% |

根本原因：GRPO 训练后，模型学会了先输出 `<think>` 推理过程再输出代码。在 lm_eval 的裸补全模式下，推理 token 会被当作代码的一部分，或提前触发 stop sequence 导致代码截断。MBPP 的变化极小（-0.20 ~ -0.40），进一步说明代码能力本身没有退化。

### 5.4 Stage2 v2（清洗后）vs Stage2 v1 对比

| 指标 | Stage2 r64 (v1) | Stage2 v2 r16 | Stage2 v2 r64 | 分析 |
|------|-----------------|---------------|---------------|------|
| humaneval | 67.68 | 67.68 | 65.85 | v2 r16 持平，v2 r64 略低 |
| mbpp | 61.80 | 61.60 | 61.80 | 基本持平 |
| gsm8k | 67.93 | 67.93 | 67.93 | 完全一致 |
| hellaswag | 67.86 | 67.91 | 67.86 | 基本持平 |
| arc_challenge | 49.15 | 49.32 | 48.81 | v2 r16 最高 |
| winogrande | 64.48 | 65.11 | 65.43 | v2 版本明显恢复 |
| mmlu | 59.70 | 59.72 | 59.74 | v2 版本略高 |
| truthfulqa_mc2 | 46.35 | 46.33 | 46.33 | 基本持平 |

关键发现：
- **WinoGrande 恢复**：Stage2 v1 r64 在 WinoGrande 上有 -0.95 的下降，而 v2 r16（-0.32）和 v2 r64（0.00）明显恢复，说明数据清洗和训练修复有效减少了对语言理解能力的干扰。
- **v2 r16 vs v2 r64**：r16 在常识推理（arc_challenge +0.43, winogrande -0.32）上略优于 r64（arc_challenge -0.08, winogrande 0.00），但 r64 在 MMLU（+0.03）和 WinoGrande（0.00）上更稳定。两者差异很小，整体表现接近。
- **通用 benchmark 上 v2 与 v1 差异极小**，说明数据清洗和训练修复主要影响的是代码任务的质量（需要结合自定义评测验证），对通用能力没有显著影响。

### 5.5 LoRA rank 对比（r16 vs r64）

在 v2 版本中，r16 和 r64 的通用 benchmark 表现非常接近：

| 指标 | v2 r16 | v2 r64 | 差异 |
|------|--------|--------|------|
| 代码生成均分 | 64.64 | 63.82 | r16 +0.82 |
| 数学推理 | 67.93 | 67.93 | 持平 |
| 常识推理均分 | 60.78 | 60.70 | r16 +0.08 |
| 通用知识均分 | 53.02 | 53.03 | 持平 |

r16 在代码生成上略优（主要来自 HumanEval 的 67.68 vs 65.85），其余维度几乎无差异。这说明对于 3B 模型的 Stage2 微调，r16 已经足够，更大的 rank 并没有带来额外收益，反而在 HumanEval 上略有下降。

## 六、结论

1. **GRPO 训练未引入灾难性遗忘**。所有 5 个模型在 MMLU、HellaSwag、ARC-Challenge、TruthfulQA、WinoGrande 等通用 benchmark 上变化均在 ±0.5% 以内，知识储备和语言理解能力完好无损。

2. **数学推理能力一致提升**。GSM8K 在所有 Stage2 模型上均达到 67.93（+0.38），与 GRPO 强化逻辑推理的训练目标一致。

3. **lm_eval 的 HumanEval/MBPP 分数下降是评测协议不匹配所致，不代表代码能力退化**。自定义评测（使用训练格式 prompt + 结构化代码提取）显示 pass@1 保持 79.27% 不变。如需用 lm_eval 公平对比，应加上 `apply_chat_template` 参数。

4. **数据清洗和训练修复有效**。Stage2 v2 在 WinoGrande 上明显恢复（v1 -0.95 → v2 r16 -0.32 / v2 r64 0.00），说明清洗后的数据集减少了对语言理解能力的干扰。

5. **r16 足够用于 Stage2 微调**。v2 r16 和 v2 r64 在通用 benchmark 上表现接近，r16 在代码生成上甚至略优，更大的 rank 没有带来额外收益。
